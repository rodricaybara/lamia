# Decisiones de diseño

## Objetivo

Este directorio documenta las decisiones tomadas durante el desarrollo del proyecto.

Estas decisiones no forman parte de la normativa.

Su finalidad es explicar por qué se ha elegido una determinada solución técnica o funcional.

---

# Contenido

Cada decisión debe incluir:

- Fecha
- Contexto
- Problema identificado
- Alternativas consideradas
- Decisión adoptada
- Justificación
- Consecuencias

---

# Buenas prácticas

Registrar únicamente decisiones relevantes.

No utilizar este directorio como historial de cambios.

Las decisiones deben ayudar a comprender la evolución del proyecto.

Cuando una decisión quede obsoleta, debe conservarse indicando que ha sido reemplazada.
