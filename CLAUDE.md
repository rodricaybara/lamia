# Filosofía del proyecto

Este proyecto debe mantenerse en un único fichero HTML.

No dividir CSS o JavaScript en archivos separados.

No utilizar frameworks.

No añadir dependencias externas.

El código debe ser fácil de entender por un desarrollador amateur.

Antes de realizar cambios importantes:

- Explicar el motivo.
- Proponer alternativas.
- Esperar confirmación.

Si una solución es más sencilla aunque menos elegante, preferir la sencilla.

# Documentación

La documentación funcional del proyecto está en el directorio `docs`.

Antes de implementar una validación:

- consulta la documentación;
- identifica la regla aplicable;
- explica cómo la vas a implementar;
- indica en qué documento y apartado te has basado.
